# 📋 Backlog do Produto - StudyTrack

Este documento contém a lista de funcionalidades planejadas para o desenvolvimento do ecossistema StudyTrack, priorizadas conforme o Roadmap do projeto.

---

## 🔴 Alta Prioridade (Núcleo do Sistema)
*   **Autenticação de Usuário:** Sistema de cadastro, login e gerenciamento de sessões com JWT.
*   **Gerenciamento de Matérias (CRUD):** Interface para criar, visualizar, editar e excluir as disciplinas do semestre.
*   **Gerenciamento de Tarefas:** Criação de lista de afazeres vinculada às matérias cadastradas.
*   **Controle de Prazos:** Sistema de datas e notificações para entregas de trabalhos e provas.

## 🟡 Média Prioridade (Experiência do Usuário)
*   **Dashboard de Progresso:** Visualização gráfica da evolução dos estudos e conclusão de tarefas.
*   **Interface Responsiva (UI/UX):** Refinamento visual utilizando Flutter para garantir consistência entre Mobile e Web.
*   **Integração Backend/Frontend:** Comunicação fluida entre a aplicação Flutter e a API FastAPI.

## 🟢 Baixa Prioridade (Futuras Melhorias)
*   **Sistema de Gamificação:** Conquista de medalhas ou pontos por metas de estudo batidas.
*   **Exportação de Relatórios:** Gerar PDFs com o resumo do desempenho acadêmico mensal.

---

### Critérios de Aceite Gerais
- O código deve ser versionado via Git com mensagens de commit claras.
- As funcionalidades devem ser testadas antes da integração na branch de produção.
